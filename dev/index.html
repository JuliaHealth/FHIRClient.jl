FHIRClient

FHIRClient provides a Julia client for connecting to servers that support the Fast Healthcare Interoperability Resources (FHIR) specification and building SMART on FHIR applications.

The source code for this package is available in the GitHub repository.

FHIRClient implements the following functionality:

  1. FHIR: Fast Healthcare Interoperability Resources. Web standard for health interop.

We currently do not implement the following; however, we would like to do so in the future:

  1. CDS Hooks: Clinical Decision Support Hooks. Web standard for CDS in the EHR workflow.
  2. US Core Data Profiles: FHIR data profiles for health data in the US (“core data for interoperability”).
  3. FHIR Bulk Data API Implementation Guide: FHIR export API for large-scale data access.
  4. SMART App Launch: User-facing apps that connect to EHRs and health portals.
  5. SMART Backend Services: Server-to-server FHIR connections.
